# Deploy the easiest path

The simplest route is to use Foundry Agent Service and deploy the smallest version of the idea that still feels real.

## Recommended path

1. Finish the local proof of concept.
2. Choose the single most visible demo flow.
3. Deploy to Foundry Agent Service.
4. Verify the output with one real prompt.

## Success criterion

The deployed version should be easy to explain, easy to refresh, and easy to show to a live audience.

## When you outgrow this deployment

For richer orchestration, Azure-platform deployment patterns, governance,
and other directions beyond the spark, see `docs/05-going-further.md`.
