# Agent Framework next-level path

This path is for developers who want to move beyond the starter demo and build a more structured, code-first experience.

## Why use Microsoft Agent Framework here

Microsoft Agent Framework is a good fit when the team wants to:
- orchestrate multiple steps or agents in code
- wire tools and model calls more explicitly
- experiment with richer local flows before deploying
- keep the prototype close to a real application architecture

## Recommended use for this repo

Use the starter kit first for the quick demo. Then move to Agent Framework when you want:
1. more control over orchestration
2. a reusable app pattern instead of a prompt-only flow
3. a better bridge into a longer-running developer prototype

## Suggested progression

1. Start with the simple Foundry sample.
2. Add a small Agent Framework-based flow in your own branch.
3. Keep the same user story, but make the implementation more explicit and reusable.

## Good fit for this repo

This is the right place for teams who want a serious developer path without losing the lightweight spirit of the starter kit.
